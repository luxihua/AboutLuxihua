이미지 검색 기능
Spring Boot: RESTful API 서버를 구축하여 이미지 검색 요청을 처리하는 엔드포인트 제공.

추천 관광지 저장 및 조회 기능
Spring Boot: 추천 관광지 조회 및 저장을 위한 REST API 제공.
MariaDB: 관광지 정보, 조회 기록을 저장하는 데이터베이스 연동.
JPA+QueryDSL: 데이터베이스의 효율적인 접근을 위한 ORM으로 MariaDB와 연동.
데이터베이스 성능 문제
문제 인식
프로젝트 초기 단계에서 JPA의 기본 제공 쿼리 메서드를 사용해 MariaDB와 상호작용하였습니다. 하지만 추후 인근 관광지 추천 기능 추가를 고려하다보니 다수의 쿼리 호출이 예상되었고 재사용성이 떨어지는 문제가 발생한다는 점을 인지하였습니다. 뿐만 아니라 카테고리와 이미지 테이블 간 매핑 단계에서 단순 오타로 인한 컴파일 오류 문제를 해결하느라 상당히 많은 시간을 소요하였습니다.
문제 해결
QueryDSL로 쿼리를 작성하도록 전체 리팩토링을 진행했습니다. 이를 통해 무엇보다 코드 재사용성이 증대하였습니다. 다른 도메인에서 검색 메서드를 활용할 때 단순히 해당 메서드를 호출함으로써 이미지 조회가 가능해졌고, 가독성 또한 증가하였습니다.
Tour API 이미지 제공 문제
문제 인식
추천 관광지의 인근 명소를 제공하는 기능을 구현하면서 정보 제공에 있어서 신뢰성이 높은 Tour API을 활용하였지만, Tour API에서 제공하는 이미지가 Null값이 경우가 훨씬 많아 다른 이미지 제공이 가능한 API를 끌어와 대체하는 작업이 필요하다는 점을 인지하였습니다.
문제 해결
kakao에서 제공하는 다음(daum) 검색 API의 코드를 일부 뜯어 image 값만 가져오도록 분리하여 Tour API의 ImageSearch 메서드에 적용하였습니다. 동시에 두 개의 API를 로드하다 보니 서버 응답 시간이 길어지자, Tour API에서 null 값을 리턴 받을 경우에만 daum API를 호출하도록 조건문을 추가했습니다. 또한 kakao api는 여러 이미지가 Map 형식으로 제공되었기에 이를 배열 형식으로 변환한 후 가장 파일 크기가 적은 첫 번째 이미지를 추출하여 제공하도록 수정하였습니다.

가상결제 문제
문제 인식
결제방식의 경우 Kakao API를 사용하는 방법이 가장 유력했으나, 사업자등록증이 필요한 관계로 결제기술 도입이 불가했습니다.
문제 해결
카드 등록의 경우, 별도의 인증 과정 없이 가상 등록이 가능하도록 구현하였습니다. 실질적인 결제는 0원으로 처리하여 결제 기술 도입을 피하였습니다.
Google Maps API 연동 문제
문제 인식
실물 헬멧 자판기 개발이 현실적으로 불가했기에, 가상으로 자판기를 인천 내에 배치한 후, Google Maps API를 연동하여 자판기 배치 좌표를 중심으로 현 위치를 표시하는 과정 에서 CORS문제가 발생했습니다.
문제 해결
1. HTTP 응답 헤더인 Access-Control-Allow-Origin를 설정하기
-> 구글의 Places API에서 제공하는 API를 사용하고 있기때문에 직접 서버를 제어할 수 없었습니다.

2. 직접 서버를 구축하여 클라이언트->서버 요청이 아닌, 서버->서버 요청으로 헤더를 추가해 CORS정책을 적용되지 않게하기.
( 설명에 따르면 CORS는 브라우저의 스크립트에서 시작된 통신에 제한을 하는 것 이므로 서버 간 통신에는 적용되지 않는다는 것을 알 수 있었습니다. )
-> 시간 및 기술역량이 부족하여 서버구축을 하여 작업을 진행할 수 없었고, 당장의 로컬환경에서 해결할 방법을 찾아야 했습니다.
=> 구글링을 통해 CORS 관련 자료를 찾다보니,프록시 서버를 이용하는 방법을 알아보게 되었습니다.
프록시 서버를 이용하면, 클라이언트-프록시서버-API서버 방식으로 프록시서버의 중계를 통해 통신을 하게된다는 점을 알게 되었고
https://cors-anywhere.herokuapp.com/corsdemo 라는 프록시 서버 사용을 시도했습니다.
해당 서버를 적용하자 HTTP 응답 헤더에 Access-Control-Allow-Origin을 설정해 응답이 가능해졌고, 이를 통해 CORS 에러를 해결할 수 있었습니다.
